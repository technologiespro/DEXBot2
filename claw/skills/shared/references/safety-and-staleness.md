# Safety and Staleness Labels

- `current`: primary source surfaces for the current release, when present in the repo.
- `reference-only`: presentation reference material and any presentation examples derived from it.
- `historical-only`: legacy archive material or older reference material that should not be treated as current guidance.

Rules:
- Treat `current` files as the only place to make operational claims.
- Treat `reference-only` material as layout, tone, and framing guidance only.
- Treat `historical-only` material as stale until explicitly relabeled by the maintainer.
- If a file mixes current and historical material, split or relabel it before reuse.
