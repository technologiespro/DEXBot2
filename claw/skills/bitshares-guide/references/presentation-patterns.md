# Presentation Patterns

Structure-only guidance for the guide branch.

## Section Order

1. Hero banner / top nav
2. `BitShares Overview`
3. `BitShares Network Metadata`
4. `Configuration Reference`
5. `Resources & Links`

## Cues

- Keep the header fixed, concise, and navigation-first.
- Make the hero feel polished and public-facing, not operational.
- Use `BitShares Network Metadata` as a compact framing section, not a source-of-truth dump.
- Keep `Configuration Reference` illustrative and reference-driven.
- End with `Resources & Links` as a footer-style link grid.
